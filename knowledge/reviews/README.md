# Reviews

This directory stores code review findings and checklists.

## Purpose

Reviews capture patterns and issues found during code reviews. They help build a project-specific knowledge base of common issues to check for.

## File Naming

Use descriptive names that indicate the review focus:
- `api-endpoint-review.md`
- `authentication-flow-audit.md`
- `performance-review-2026-01.md`

See `../TEMPLATES.md` for the review file template.
